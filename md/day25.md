# 25강 - 클래스 이론

## 클래스
- 공통요소가 모여있는곳
- 주어이다.
    - 원숭이가 바나나를 먹는다
    - Monkey.eat("바나나") : 주어 동사 목적어
    - 클래스는 문장의 맨 앞에 등장
    - 앞글자를 항상 대문자로 작성한다
- 타입이다.
    - 공통요소가 있는 것들을 여러개 선언하면 비효율적이기 때문에, 클래스에 한 번씩 만 선언하고 해당 필드를 가져다 쓴다.
    - 대신 해당 필드에 접근하기 위해서는 그 클래스의 타입으로 선언되어야한다.

### 클래스 선언
```
class 클래스명:
    필드(객체,메소드)
```

## 객체 (instance variable)
- 클래스는 추상적인 개념이고, 이 추상적인 개념을 실제화 시켜야지만 사용할 수 있다.
- 따라서 추상적인 개념을 실제화(구제화) 시켜놓은 것을 객체 라고한다.
- 클래스: 도형 , 객체 : 세모

## 객체화 (instance)
- 추상적인 개념을 구체화시키는 작업
- 객체명 = 클래스명()

## 생성자 
- 클래스명()을 생성자라고 하며, 이름뒤에 소괄호가 있기 떄문에 메소드의 기능과 똑같다. 리턴이 없기 때문에 메소드라 부르지않고 생성자라고 부른다
- 대문자로 시작하는지 확인 !
- 클래스에 있는 필드를 메모리에 할당해주고 그 필드의 주소값을 가지고 돌아와야 하기 떄문에, 개발자가 직접 리턴값을 작성하면 안된다.

### 생성자의 역할
1. 해당 필드를 메모리(RAM)에 할당해줌.
    - 메모리에 할당된 주소값을 리턴해야함
2. 초기화
